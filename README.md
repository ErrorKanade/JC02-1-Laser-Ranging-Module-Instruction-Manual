
 JC02-1 激光测距模组扩展库与位移控制集成系统 (基于 Arduino Nano)

学习工业级 JC02-1 远程激光测距模组 (200m-2000m) 出现了一系列故障原因。
本次设计重点解决了在资源受限（2KB SRAM）平台上，软件串口（SoftwareSerial）与 I2C OLED 屏幕、PID 控制器因中断阻塞和内存溢出导致的系统卡死、丢包、持续触发盲区报警（`607` 错误码）等核心工程问题。

---

  模组核心技术指标 (摘自说明书)

在进行硬件开发前，必须严格对齐以下光学与电气特性：
 光学波长：905nm 半导体激光（不可见光，脉冲能量高达 1000nJ）。
 物理盲区与精度：分辨率 `0.3m`，近距离死区测距无法出数；200米内精度 `0.6m`。
 激光发散角：`3 mrad`（毫弧度），长距离光斑会变大。
 严格供电区间：`+3.3V ~ +3.8V`（发射瞬间瞬态脉冲电流极大，严禁直接取自 Nano 的 3.3V 引脚）。
 数据单位：回传协议中倒数第三字节（单位字节）为 `01` 时，代表单位为 `0.1m`（分米）。

---

开发遇见的问题与分析 

在主控集成过程中，通常会遭遇以下系统性崩溃（Systemic Crashes），本驱动已对其进行了针对性优化：

 1. 串口没有显示，OLED 也不亮（程序启动卡死）
 原因分析：Arduino Nano 仅有 2KB SRAM。标准 `Adafruit_SSD1306` 库为了支持 128x64 图像会强行开辟 1024 字节 的静态显存缓冲区。加上 `SoftwareSerial` 缓冲区和串口打印字符串，导致内存瞬间溢出（OOM），进 `loop()` 前系统即锁死。
 解决对策：对所有串口打印的静态字符串强制使用 `F()` 宏（将其常驻 Flash 减少内存占用），并拉长 OLED 刷新周期，防止 I2C 库无超时挂死总线。

 2. 持续高频输出 `[ALARM] 607` 错误码
 原因分析：`607`（Hex: `02 5F`）是该模组判定“无法捕获有效回波”的官方错误码。常见于：
  1. 供电不足导致激光管点火失败。
  2. 模组贴地放置，3mrad 发散角导致桌面近场漫反射信号过载（过曝）。
  3. 目标距离小于 0.3~1.5 米的物理盲区。
 解决对策：更换独立 DCDC 稳压电源（如 3.7V 锂电池或独立稳压模块），确保光轴架高离地至少 15cm 以上，并对准 5 米外的高反射率目标。

 3. 数据包错位与乱码导致的解析失败
 原因分析：OLED 刷屏函数 `display.display()` 运行一次需阻塞中断约 33~40ms。在此期间，Nano 的 CPU 无法响应 `SoftwareSerial` 的软中断，导致 Nano 仅 64 字节的硬件串口缓冲区发生雪崩式溢出，数据包断裂、错位。
 解决对策：废弃 `readBytes` 等阻塞读取函数，改用低开销状态机逐字节流式解析，并采用异步时间片轮询，将 OLED 刷新频率压低至 `500ms~1000ms`。

  硬件接线引脚分配 (Nano 方案)

由于激光模组瞬间功耗较高，必须做供电隔离，主控与外设需强制共地：

| 激光模组引脚 | 功能描述 | Arduino Nano 引脚 | 外部供电网络 |
| :---: | :---: | :---: | :---: |
| VCC | 供电 (3.3V-3.8V) | 悬空 (严禁接Nano引脚) | 外部稳压电源正极 (推荐 3.5V-3.7V) |
| GND | 系统地 | GND | 外部稳压电源负极 (系统共地) |
| UART TX | 模组数据发送 | Pin 2 (Software RX) | - |
| UART RX | 模组指令接收 | Pin 3 (Software TX) | - |
| OLED SDA | I2C 数据总线 | A4 | - |
| OLED SCL | I2C 时钟总线 | A5 | - |
